# 智能指纹门锁系统 v2.0 — 完整方案 (Nano + 2.2" TFT)

> **MCU**: Arduino Nano (ATmega328P, 5V 逻辑, AVR, 16 MHz)
> **屏幕**: 2.2" SPI ILI9341 240×320 彩色 TFT (横屏使用 320×240)
> **传感器/外设**: TM1026M 指纹、KY-038 麦克风、MAX9812 声音波形、ISD1820 录音 + 喇叭、5V 继电器、共阴 RGB、无源蜂鸣器

---

## 0. v2.0 新特性 (相对 0.96 OLED 版)

| 类别 | 升级内容 |
|:--|:--|
| 屏幕 | 0.96″ 单色 OLED → 2.2″ 彩色 TFT (面积 ×7) |
| 桌面宠物 🐱 | 待机屏左右走动的小猫, **4 种情绪** (开心/睡觉/兴奋/伤心), 根据使用频率自动切换 |
| 个性化 | 每个指纹 ID 独立 **姓名 / RGB 颜色 / 旋律**, 解锁时显示彩色头像 |

| 暴力锁定 | 连续 3 次失败 → **30 秒锁定 + 红灯爆闪报警** |
| 解锁日志 | EEPROM 滚动存最近 10 次开锁记录 |
| 门铃模式 | 声音唤醒后 15s 内无人放指纹 → 自动播 ding-dong + 屏幕"Visitor" |
| 静音模式 | 串口 `silent` 切换, 夜间不扰民 |
| **Bug 修复** | 解决 0.96 版 **D11 RGB_B 与 tone() Timer2 冲突** 导致的蓝灯抖动 |
| **Bug 修复** | 录入流程的 **"等待抬手指"** 死循环 → 加 8s 超时 |

## 2. 引脚分配表 (v2.0)

| Nano 引脚 | 类型 | 连接组件 | 说明 |
|:---:|:---:|:---:|:---|
| **D2** | 数字 | TM1026M TX (绿) | SoftwareSerial RX |
| **D3** | PWM | TM1026M RX (白) | SoftwareSerial TX, **经分压 5V→3.3V** |
| **D4** | 数字 | ISD1820 PLAYE | 边沿触发播放 |
| **D5** | PWM | **RGB-B** | (从原 D11 迁移过来) |
| **D6** | PWM | 继电器 IN | 控制电磁锁 |
| **D7** | 数字 | KY-038 DO | 数字声音触发 (LOW 有效) |
| **D8** | 数字 | **蜂鸣器 S** | tone() 输出, **新位置** |
| **D9** | PWM | RGB-R | 共阴 220Ω 限流 |
| **D10** | PWM | RGB-G | 共阴 220Ω 限流 |
| **D11** | SPI | **TFT MOSI** | SPI 数据 |
| **D12** | SPI | TFT MISO | 不接也行 (TFT 不需要回读) |
| **D13** | SPI | **TFT SCK** | SPI 时钟 (板载 LED 让位) |
| **A0** | 模拟 | MAX9812 OUT | 10-bit ADC 波形采样 |
| **A1** | 数字 | **TFT CS** | 片选 |
| **A2** | 数字 | **TFT DC** | 数据/命令 |
| **A3** | 数字 | **TFT RST** | 复位 |
| **A4 / A5** | I²C | (空闲) | 旧 OLED 用, 现可空 |
| **5V** | 电源 | 全部 5V 模块 | TFT/继电器/KY-038/ISD1820/蜂鸣器 |
| **3V3** | 电源 | TM1026M / MAX9812 | 3.3V 模块, 由 Nano 板载 LDO 输出 |
| **GND** | 电源 | 全部模块共地 | **必须共地** |

> ⚠️ **关键变更**:
> 1. RGB_B 从 D11 → **D5**, 蜂鸣器从 D5 → **D8**, 这样 SPI 总线 (D11/D12/D13) 让给 TFT, 同时彻底解决了 `tone()` 与 RGB_B 共用 Timer2 导致的蓝灯抖动 bug。
> 2. SPI 总线在 Nano 上是固定引脚, 改不了。
> 3. **运行时 USB 串口仍可用** (D0/D1 不冲突), 烧录时如果调试串口连着 USB-TTL 也没事。

## 3. 详细接线图

### 1. ILI9341 2.2" SPI TFT
```
TFT 模块         Nano
────────         ───────
VCC      ──────► 5V        (模块板载 3.3V LDO, 接 5V 安全)
GND      ──────► GND
CS       ──────► A1
RESET    ──────► A3
DC / RS  ──────► A2
SDI/MOSI ──────► D11
SCK      ──────► D13
LED      ──────► 5V        (背光直接 5V; 想 PWM 调亮度可接 D8 但占用蜂鸣器)
SDO/MISO ──────► (不接)    (TFT 不回读, 省一根)
```

### 2. 指纹模块 TM1026M (与 v1 完全相同)
见 v1 章节; **D3 → RX 必须 1KΩ + 2KΩ 分压到 3.3V**。

### 3. 无源蜂鸣器
```
Buzzer       Nano
──────       ───────
S(+)   ────► D8        (变了, 不再是 D5)
GND    ────► GND
```

### 4. RGB LED (共阴极)
```
RGB              Nano
───              ───────
R ──[220Ω]────►  D9   (PWM)
G ──[220Ω]────►  D10  (PWM)
B ──[220Ω]────►  D5   (PWM, 变了)
GND ──────────►  GND
```

### 5. 继电器 / KY-038 / MAX9812 / ISD1820
与 v1 完全相同, 不再赘述。

## 4. EEPROM 数据布局 (1024 B)

```
0x000  4B   Magic 0xA5C30001  (校验是否已初始化)
0x004  1B   unlockCount low
0x005  1B   unlockCount high
0x006  1B   themeId
0x007  1B   silentMode
0x020  128B Users[8] 每个 16B
              [+0]  active=0xAA
              [+1..+9]  name (8 char + null)
              [+10..+12]  R/G/B
              [+13]  melodyId (0-3)
0x0A0  576B Messages[6] 每个 96B
              [+0]  active=0xAA
              [+1]  targetId (0=群发)
              [+2]  readFlag
              [+3]  flags (bit0=置顶)
              [+4..+5]  createdHour
              [+6..+7]  ttlHours (0xFFFF=永久)
              [+8..+87]  text (80B null-terminated)
0x2E0  4B   petXP / petLastInteractHour
0x2F8  60B  Log[10] 每个 6B (id + uptimeHour)
0x334  1B   logHead
0x340-0x3FF reserved (~190B 余量)
```

总占用 ≈ 832B / 1024B (剩 192B 给未来扩展)。

## 5. 串口命令 (波特率 9600)

| 命令 | 说明 |
|:--|:--|
| `e` | 进入指纹录入模式 (后续按提示放手指 2 次) |
| `d` | 清空所有指纹 |
| `u <id> <name> <r> <g> <b> <mel>` | 设置某 ID 用户的姓名/颜色/旋律 (例: `u 1 Yang 0 200 80 1`) |
| `ul` | 列出所有用户 |
| `m <text>` | 添加群发留言 |
| `m@<id> <text>` | 定向留言给特定 ID (例: `m@2 钥匙在抽屉`) |
| `m! <text>` | 添加置顶留言 |
| `ml` | 列出所有留言 |
| `md <idx>` | 删除指定留言 |
| `mc` | 清空所有留言 |
| `log` | 查看最近 10 次解锁记录 |
| `silent` | 切换静音模式 |
| `stats` | 查看统计 (开锁次数 / 运行小时数 / 宠物经验) |
| `help` | 列出帮助 |

> 留言文本最大 80 字符, 暂不支持中文 (用 ASCII)。

## 6. 桌面宠物 🐱

待机屏右下角有一只小猫:
- **happy** 默认状态, 睁着圆眼, 摇尾巴, 在屏幕右侧来回走
- **excited** 解锁后短暂触发, 星眼 + 心心泡泡
- **sad** 解锁失败 / 8 小时无人理触发, 耳朵下垂 + 流泪
- **sleepy** 24 小时没人理触发, 灰色 + Z 字, 静止不动

每次成功解锁 +10 经验, 经验存 EEPROM 重启不丢。

## 9. 快速上手

### Step 1 — 接线
按 §3 接线表组装。**最容易踩坑**: TFT 的 CS/DC/RST 是 A1/A2/A3, 不是 D 引脚 (Nano 的 A 脚也能当数字 IO)。

### Step 2 — 录 ISD1820 语音
单独给 ISD1820 接 5V/GND/喇叭, 按住 REC 录"欢迎回家"(≤10s), 松开 REC, 按 PLAYE 验证。

### Step 3 — 上传代码
1. Arduino IDE → 开发板: **Arduino Nano**, 处理器: **ATmega328P** (老板子选 Old Bootloader)。
2. 安装 §8 三个库。
3. 打开 `fingerprint_door_lock.ino`, 上传。
4. 期望开机看到: 扫描线 → 锁图标 → "Smart Door Lock" + 五声上行音, 然后进入 LOCKED 待机屏。

### Step 4 — 录入指纹
串口监视器, 9600:
```
> e
Enroll ID=1
> [放手指 → 移开 → 再放同一手指]
Enroll OK ID=1
```

### Step 5 — 配置个性化 + 留言
```
> u 1 Yang 0 200 80 1
> u 2 Mom  255 100 100 0
> m Welcome home everyone!
> m@1 Yang, your package arrived
```
重启后所有配置都还在 (EEPROM)。

### Step 6 — 联调测试
1. 拍手 → 进入 WAKE 动画。
2. 已录入手指 → 个性化欢迎 + 留言弹出 + 5s 后自动锁。
3. 错指纹 3 次 → 暴力锁定 30s。
4. 拍手后 15s 不动 → 门铃模式。
5. `silent` 命令切静音。

## 11. 常见问题

| 现象 | 原因 / 排查 |
|:--|:--|
| TFT 全白 / 没图像 | 检查 CS/DC/RST 是否接到 A1/A2/A3; 电源 5V 是否稳定 |
| TFT 颜色反了 | 调用 `tft.invertDisplay(true)` (代码暂未启用) |
| 指纹模块"未找到" | TX/RX 是否交叉; D3 → RX 必须分压; 测量 3.3V 供电 |
| 不会被声音唤醒 | KY-038 板载电位器调灵敏度; 阈值改 60-150 |
| 编译报错 "out of memory" | 库没装全; 或 Nano flash 不够, 删掉部分动画 |
| 上传 Sketch too big | 同上, 优先删 `drawSparkle` 烟花和 sin/cos 动画 |
| 蓝灯有抖动 (老问题) | v2 已修复 (RGB_B 移到 D5) |
| RGB 颜色不对 | 共阳极板把 `RGB_COMMON_ANODE` 设 `true` |
| 留言不显示 | 检查 `ml` 是否有数据; 留言文本只支持 ASCII |
| 桌面宠物总是 sad | 没人解锁超过 8 小时; 解锁一次就会变回 happy |

## 13. 后续可拓展方向 (v3 候选)

不需新硬件, 纯软件:
- **敲击密码备用钥匙** (摩斯码)
- **打卡热力图** (一周 7 格点亮)
