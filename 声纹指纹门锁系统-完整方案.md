# 声纹指纹门锁系统 — 完整方案 (Arduino Nano ATmega328P 版)

> **MCU**: Arduino Nano (ATmega328P, 5V 逻辑, AVR, 16 MHz)
> **特点**: 经典 5V Nano, PWM 引脚仅 D3/D5/D6/D9/D10/D11, 用 SoftwareSerial 接指纹
> **传感器**: TM1026M 指纹、KY-038 麦克风、MAX9812 声音波形、SSD1306 OLED、ISD1820 录音、5V 继电器、共阴 RGB、无源蜂鸣器

---

## 一、系统工作流程

```
待机 (OLED 锁屏 + 红色呼吸 + 滚动横幅)
   │  KY-038 / MAX9812 检测到声音
   ▼
唤醒动画 (麦克风 + 声波扩散) + RGB 蓝 + "叮咚"
   │
   ▼
等待指纹 (扫描动画 + 双向流动进度条 + 蓝色脉冲)
   │
   ├─ 匹配成功 ──► 开锁动画(放射线 + 烟花 + WELCOME)
   │              RGB 绿色渐暗 + 继电器吸合
   │              成功旋律 (每 5 次解锁触发"超级玛丽"彩蛋)
   │              ISD1820 播放"欢迎回家"
   │              5 秒后自动上锁
   │
   └─ 匹配失败 ──► 抖动 X 动画 + 警告音 + RGB 红
```

## 二、引脚分配表 (Nano ATmega328P)

| Nano 引脚 | 类型 | 连接组件 | 说明 |
|:---:|:---:|:---:|:---|
| **D2** | 数字 | 指纹 TX (绿) | SoftwareSerial RX |
| **D3** | PWM | 指纹 RX (白) | SoftwareSerial TX, **经分压 5V→3.3V** |
| **D4** | 数字 | ISD1820 PLAYE | 边沿触发播放 |
| **D5** | PWM | 蜂鸣器 S | 输出方波 |
| **D6** | PWM | 继电器 IN | 控制电磁锁 |
| **D7** | 数字 | KY-038 DO | 数字声音触发 |
| **D9** | PWM | RGB-R | 真彩呼吸 (220Ω) |
| **D10** | PWM | RGB-G | 真彩呼吸 (220Ω) |
| **D11** | PWM | RGB-B | 真彩呼吸 (220Ω) |
| **A0** | 模拟 | MAX9812 OUT | 10-bit ADC 波形采样 |
| **A4 / SDA** | I²C | OLED SDA | I²C 数据 |
| **A5 / SCL** | I²C | OLED SCL | I²C 时钟 |
| **5V** | 电源 | 全部 5V 模块 | OLED/继电器/KY-038/ISD1820 等 |
| **3V3** | 电源 | TM1026M / MAX9812 | 3.3V 模块 (Nano 板载 LDO 输出) |
| **GND** | 电源 | 全部模块共地 | **必须共地** |

> ⚠ **关键约束**:
> 1. AVR Nano 只有 D3/D5/D6/D9/D10/D11 是 PWM, 所以 RGB 接到 **D9/D10/D11** 才能呼吸。
> 2. 只有一组硬件串口 (D0/D1) 用于 USB 调试, 指纹必须用 `SoftwareSerial` (D2/D3)。
> 3. Nano 是 5V 逻辑, 但 TM1026M 是 3.3V 模块: TX→D2 直连 OK, **D3→RX 必须经分压**。

## 三、详细接线图

### 1. OLED 0.96" (I²C / SSD1306)

```
OLED       Nano
────       ───────
VCC  ───►  5V       (SSD1306 模块通常接 5V 也可工作)
GND  ───►  GND
SDA  ───►  A4
SCL  ───►  A5
```
默认地址 `0x3C`, 部分模块为 `0x3D` (代码已自动尝试两个)。

### 2. 指纹模块 TM1026M (6Pin, UART, 3.3V)

```
TM1026M           Nano
─────────         ───────
VCC (红)   ─────► 3V3       (TM1026M 工作 3.3V, 切勿接 5V!)
GND (黑)   ─────► GND
TX  (绿)   ─────► D2        (3.3V → 5V 直连, AVR 输入门槛 ~3V 可识别)
RX  (白)   ─────► [分压]──► D3
TOUCH/IRQ  ─────► (可不接)
3V3-OUT    ─────► (悬空)
```

**5V → 3.3V 分压电路 (RX 必须做)**:

```
   D3 ──────┬──[1KΩ]──┬──► 模块 RX
            │          │
            │       [2KΩ]
            │          │
            └──────────┴──► GND
```

> Vout = 5V × 2K / (1K+2K) ≈ 3.3V ✅
> 若手头没有电阻, 可用一颗 74HC125/双向电平转换板替代。

默认波特率 57600, 协议兼容 AS608, 直接用 `Adafruit_Fingerprint`。

### 3. 无源蜂鸣器

```
Buzzer       Nano
──────       ───────
S(+)   ────► D5 (PWM)
VCC    ────► 5V         (有源板使用, 无源两脚版可不接)
GND    ────► GND
```

### 4. 1 路继电器 (5V)

```
继电器 (输入)        Nano
────────────        ───────
VCC   ────────────► 5V
GND   ────────────► GND
IN    ────────────► D6

继电器 (输出)        电磁锁回路
────────────        ─────────
COM   ────────────► 12V 适配器 (+)
NO    ────────────► 电磁锁 (+)
电磁锁 (-) ────────► 12V 适配器 (-)
```

```
续流二极管保护:
   电磁锁(+) ──┬── 1N4007 (负极朝 +) ──┬── 电磁锁(-)
               │                       │
               └───────────────────────┘
```

> 部分继电器是 **低电平触发**, 若你接好后发现继电器一直吸合 / 逻辑反了, 把代码 `digitalWrite(RELAY_PIN, HIGH/LOW)` 互换即可。

### 5. ISD1820 录音模块 + 喇叭

```
ISD1820        Nano
───────        ───────
VCC      ────► 5V
GND      ────► GND
PLAYE    ────► D4          (高脉冲 → 完整播放一次)
REC      ────► (录音时按板载按键)
SP+ / SP- ──► 接 8Ω 小喇叭
```

### 6. KY-038 数字声音检测

```
KY-038       Nano
──────       ───────
+ (VCC) ───► 5V
G (GND) ───► GND
DO      ───► D7       (检测到声音 = LOW)
AO      ───► (不接, 由 MAX9812 提供模拟波形)
```
板载电位器先转到中间, 调试时再细调。

### 7. MAX9812 声音波形

```
MAX9812      Nano
───────      ───────
VCC   ─────► 3V3      (避免与 KY-038 干扰, 用 3V3 更干净)
GND   ─────► GND
OUT   ─────► A0
```

### 8. RGB LED (共阴极, 默认)

```
RGB              Nano
───              ───────
R ──[220Ω]────►  D9  (PWM)
G ──[220Ω]────►  D10 (PWM)
B ──[220Ω]────►  D11 (PWM)
GND ──────────►  GND
```
共阳极: 公共脚接 5V, 把代码 `RGB_COMMON_ANODE` 设为 `true`。

### 9. 2.2" 串口触摸屏

> 本方案不使用 (引脚优先 OLED), 后续升级可替换。

## 四、供电方案

```
[12V/2A 适配器] ──┬─► 12V 电磁锁 (经继电器 COM/NO)
                  │
                  └─► [12V→5V 降压模块] ──► Nano VIN (7-12V) 或 5V Pin
                                            └─► 全部 5V 模块

Nano 板载 LDO ──► 3V3 ──► TM1026M / MAX9812
```

> **共地是稳定运行的关键**。USB 调试时也要把外部 12V 系统的 GND 与 USB GND 连起来。

## 六、操作步骤

### Step 1: Arduino IDE 准备

1. 安装 Arduino IDE 1.8.x 或 2.x。
2. `工具 → 开发板 → Arduino AVR Boards → Arduino Nano`。
3. `工具 → 处理器 → ATmega328P` (老版烧录器选 "ATmega328P (Old Bootloader)")。
4. 端口选对应 `/dev/cu.usbserial-XXXX` (Mac) 或 `COMx` (Win)。

### Step 2: 安装库 (库管理器)

| 库名 | 作者 | 说明 |
|:--|:--|:--|
| `Adafruit SSD1306` | Adafruit | OLED 驱动 |
| `Adafruit GFX Library` | Adafruit | 图形基础 |
| `Adafruit Fingerprint Sensor Library` | Adafruit | 指纹协议 |

> `SoftwareSerial` 与 `Wire` 是 IDE 自带, 无需安装。

### Step 3: 先单独录制 ISD1820 语音

不要插 Arduino, 单独给 ISD1820 接 5V/GND/喇叭:

1. 按住 **REC**, 对麦克风说 **"欢迎回家"** (≤10 秒)。
2. 松开 REC。
3. 按 **PLAYE** 验证播放正常。

> ISD1820 只能存 1 段, 推荐录"欢迎回家", 失败提示由蜂鸣器旋律负责。

### Step 4: 按接线表组装

建议顺序 (先低风险后高风险):

1. **OLED** → 上电后能看到开机动画 (扫描线 + 锁图标 + 星点)。
2. **蜂鸣器** → 听到开机旋律 C-E-G-C-E。
3. **指纹模块** (注意分压) → 串口窗口打印 `指纹模块: OK`。
4. **KY-038 + MAX9812** → 拍手能触发"声音唤醒"。
5. **RGB LED** → 待机时红色呼吸, 唤醒时蓝色脉冲。
6. **继电器** → 测试匹配后听到"咔哒"吸合。
7. **ISD1820** → 解锁后播报"欢迎回家"。

### Step 5: 上传代码

打开 `fingerprint_door_lock.ino` → 点 **上传**。
**坑提示**: 老 Nano 通常用 CH340 USB 串口芯片, Mac 上要先装 CH340 驱动; 上传报错就把处理器切到 "ATmega328P (Old Bootloader)"。

### Step 6: 录入指纹

打开串口监视器, **波特率 9600**:

- 发送 `e` → 进入录入模式, 按提示放手指 2 次 (中间有"移开手指"提示)。
- 发送 `d` → 清空全部指纹 (谨慎!)。
- 最多可存 127 枚。

### Step 7: 联调

1. 拍手 / 说话 → 系统从待机进入唤醒。
2. 把已录入的手指放上 → 旋律响起 + RGB 绿渐暗 + 继电器开 + 喇叭播报。
3. 按错误手指 → 抖动 X 动画 + 下行警告音。
4. 等 5 秒 → 自动上锁 + 下行旋律。
5. **彩蛋**: 连续解锁 5 次, 会播放一段"超级玛丽"风格升级音 🍄。

### Step 8: 调灵敏度

- KY-038 板载电位器: 不会自动唤醒就慢慢拧 (灵敏度调高); 误触发就反方向。
- 代码 `detectSound()` 中 `peak > 100` 阈值可改 (60–200)。

## 七、OLED 动画与蜂鸣器旋律一览

| 场景 | OLED 动画 | 蜂鸣器旋律 | RGB |
|:--|:--|:--|:--|
| 开机 | 扫描线刷屏 + 锁 + 星点 + "Nano ATmega328" | C4-E4-G4-C5-E5 上行 | 关 |
| 待机 | 锁 + 呼吸圆环 + 滚动 "Speak to wake up" | — | 红色呼吸 (PWM) |
| 唤醒 | 麦克风 + 三圈声波扩散 | E5→C5 "叮咚" | 蓝色 |
| 等待指纹 | 旋转指纹纹路 + 双向流动进度条 | — | 蓝色脉冲 (PWM) |
| 成功 | 锁打开 + 12 道放射线 + 烟花星点 + WELCOME | C5-E5-G5-C6-G5-C6 上行 | 绿色渐暗 |
| **彩蛋(每 5 次)** | 同上 | 超级玛丽风格升级音 | 绿色 |
| 失败 | 左右抖动的 X + ACCESS DENIED | 500-400-300-200-200 下行警告 | 红色 |
| 已开锁 | 锁开 + 双心 + 倒计时进度条 | — | 绿色渐暗 |
| 上锁 | 回到待机 | G4-E4-C4 平稳下行 | — |
| 超时 | 回到待机 | A4-A4 短促 | — |

## 八、常见问题

| 现象 | 原因 / 排查 |
|:--|:--|
| 串口看不到输出 | 监视器波特率改 **9600**; CH340 驱动是否装好 |
| 上传报错 `avrdude: stk500_recv()` | 处理器切到 "ATmega328P (Old Bootloader)" |
| `指纹模块: 未找到` | TX/RX 是否交叉; D3 → RX **必须分压**; 测电源 3.3V |
| OLED 黑屏 | I²C 地址换 0x3D; 检查 SDA/SCL 是否反 |
| 不会被声音唤醒 | KY-038 板载电位器调灵敏度; 阈值改 60 |
| 继电器一直响 / 逻辑反 | 部分模块低电平触发, 把 `RELAY_PIN` 高低互换 |
| RGB 颜色不对 | 共阳极板把 `RGB_COMMON_ANODE` 设 `true` |
| Sketch 太大 | 若 RAM 紧张, 删掉烟花/星点动画段 |
