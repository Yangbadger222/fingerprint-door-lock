# 智能指纹门锁系统 v2.0 — 完整方案 (Nano + 2.2" TFT)

> **MCU**: Arduino Nano (ATmega328P, 5V 逻辑, AVR, 16 MHz)
> **屏幕**: 2.2" SPI ILI9341 240×320 彩色 TFT (横屏使用 320×240)
> **传感器/外设**: TM1026M 指纹、KY-038 麦克风、MAX9812 声音波形、ISD1820 录音 + 喇叭、5V 继电器、共阴 RGB、无源蜂鸣器

---

## 1. 系统状态机

```
                  ┌────────────────────────────┐
                  │            IDLE             │
                  │  (锁屏 + 桌面宠物 + 滚动留言) │
                  │     红色呼吸 (有未读=紫色)   │
                  └──────────┬──────────────────┘
                             │ 检测到声音
                             ▼
                  ┌────────────────────┐
                  │       WAKE          │ 1.5s 麦克风+声波动画
                  └──────────┬──────────┘
                             ▼
               ┌─────────────────────────────┐
               │ 有口令模板?                  │
               │  是 → PASSPHRASE (说口令验证) │
               │  否 → 直接 WAIT_FINGER       │
               └──────┬──────────────┬────────┘
                  通过 │              │ 失败
                      ▼              ▼
              ┌─────────────┐   ┌──────────┐
              │ WAIT_FINGER │   │   FAIL   │
              └──────┬──────┘   └──────────┘
                     │
          匹配成功   │   匹配失败
              ▼      │      ▼
       ┌─────────┐   │  ┌─────────┐
       │MATCH_OK │   │  │MATCH_FAIL│
       └────┬────┘   │  └────┬────┘
            ▼        │       │  失败 3 次?
       ┌─────────┐   │       ▼
       │UNLOCKED │   │  ┌──────────┐
       │ 5s 倒计时│   │  │ LOCKOUT  │ 30s
       └────┬────┘   │  └──────────┘
            ▼        │
          IDLE       │

   超时 (15s 没指纹) → DOORBELL → 播门铃 → IDLE
```

## 2. 引脚分配表 (v2.0)

| Nano 引脚 | 类型 | 连接组件 | 说明 |
|:---:|:---:|:---:|:---|
| **D2** | 数字 | TM1026M TX (绿) | SoftwareSerial RX |
| **D3** | PWM | TM1026M RX (白) | SoftwareSerial TX, **经分压 5V→3.3V** |
| **D4** | 数字 | ISD1820 PLAYE | 边沿触发播放 |
| **D5** | PWM | **RGB-B** | (从原 D11 迁移过来) |
| **D6** | PWM | 继电器 IN | 控制电磁锁 |
| **D7** | 数字 | KY-038 DO | 数字声音触发 (LOW 有效) |
| **D8** | 数字 | **蜂鸣器 S** | tone() 输出, **新位置** |
| **D9** | PWM | RGB-R | 共阴 220Ω 限流 |
| **D10** | PWM | RGB-G | 共阴 220Ω 限流 |
| **D11** | SPI | **TFT MOSI** | SPI 数据 |
| **D12** | SPI | TFT MISO | 不接也行 (TFT 不需要回读) |
| **D13** | SPI | **TFT SCK** | SPI 时钟 (板载 LED 让位) |
| **A0** | 模拟 | MAX9812 OUT | 10-bit ADC 波形采样 |
| **A1** | 数字 | **TFT CS** | 片选 |
| **A2** | 数字 | **TFT DC** | 数据/命令 |
| **A3** | 数字 | **TFT RST** | 复位 |
| **A4 / A5** | I²C | (空闲) | 旧 OLED 用, 现可空 |
| **5V** | 电源 | 全部 5V 模块 | TFT/继电器/KY-038/ISD1820/蜂鸣器 |
| **3V3** | 电源 | TM1026M / MAX9812 | 3.3V 模块, 由 Nano 板载 LDO 输出 |
| **GND** | 电源 | 全部模块共地 | **必须共地** |

> ⚠️ **关键变更**:
> 1. RGB_B 从 D11 → **D5**, 蜂鸣器从 D5 → **D8**, 这样 SPI 总线 (D11/D12/D13) 让给 TFT, 同时彻底解决了 `tone()` 与 RGB_B 共用 Timer2 导致的蓝灯抖动 bug。
> 2. SPI 总线在 Nano 上是固定引脚, 改不了。
> 3. **运行时 USB 串口仍可用** (D0/D1 不冲突), 烧录时如果调试串口连着 USB-TTL 也没事。

## 3. 详细接线图

### 1. ILI9341 2.2" SPI TFT
```
TFT 模块         Nano
────────         ───────
VCC      ──────► 5V        (模块板载 3.3V LDO, 接 5V 安全)
GND      ──────► GND
CS       ──────► A1
RESET    ──────► A3
DC / RS  ──────► A2
SDI/MOSI ──────► D11
SCK      ──────► D13
LED      ──────► 5V        (背光直接 5V; 想 PWM 调亮度可接 D8 但占用蜂鸣器)
SDO/MISO ──────► (不接)    (TFT 不回读, 省一根)
```

### 2. 指纹模块 TM1026M (与 v1 完全相同)
见 v1 章节; **D3 → RX 必须 1KΩ + 2KΩ 分压到 3.3V**。

### 3. 无源蜂鸣器
```
Buzzer       Nano
──────       ───────
S(+)   ────► D8        (变了, 不再是 D5)
GND    ────► GND
```

### 4. RGB LED (共阴极)
```
RGB              Nano
───              ───────
R ──[220Ω]────►  D9   (PWM)
G ──[220Ω]────►  D10  (PWM)
B ──[220Ω]────►  D5   (PWM, 变了)
GND ──────────►  GND
```

### 5. KY-038 麦克风声音检测模块 (数字触发)
```
KY-038             Nano
──────             ───────
VCC      ──────►  5V
GND      ──────►  GND
DO       ──────►  D7        数字输出, 有声音时 LOW

(AO 不接, 本系统用 MAX9812 做模拟采样)
```
> ⚠️ 板载蓝色电位器调节触发灵敏度, 顺时针更灵敏。建议先调到中间位置。

### 6. MAX9812 声音波形模块 (模拟采样)
```
MAX9812            Nano
───────            ───────
VCC      ──────►  3.3V      ⚠️ 必须 3.3V, 不是 5V!
GND      ──────►  GND
OUT      ──────►  A0        10-bit ADC (0-1023)
```
> ⚠️ 供电接 3.3V (Nano 板载 LDO 输出)。接 5V 不烧但波形削顶失真。

### 7. ISD1820 语音录放模块
```
ISD1820            Nano          喇叭
──────             ───────       ─────
VCC      ──────►  5V
GND      ──────►  GND
PLAYE    ──────►  D4            边沿触发播放 (上升沿)
SP+      ──────────────────────► 喇叭 +
SP-      ──────────────────────► 喇叭 -
```
> 喇叭: 8Ω 0.5W 小喇叭。录音用手动按键 (按住 REC → 对着麦克风说"欢迎回家"≤10s → 松开), 录音存芯片内部, 断电不丢。

### 8. 继电器模块 (控制电磁锁)
```
Relay              Nano          电磁锁回路
─────              ───────       ──────────
VCC      ──────►  5V
GND      ──────►  GND
IN       ──────►  D6            高电平触发

COM      ──────►  12V 电源正极
NO       ──────►  电磁锁正极    通电时 COM-NO 接通
(电磁锁负极) ──►  12V 电源负极
```
> ⚠️ 电磁锁两端并联 1N4007 续流二极管 (阴极接正极), 防继电器释放时反向电动势。
> 没有电磁锁时, 用万用表蜂鸣档测 COM-NO 通断, 或听 "咔嗒" 声确认。

### 9. TM1026M 指纹传感器 (3.3V UART)
```
TM1026M            Nano
───────            ───────
VCC (红)  ──────►  3.3V          ⚠️ 必须 3.3V, 接 5V 会烧!
GND (黑)  ──────►  GND
TX  (绿)  ──────►  D2            SoftwareSerial RX
RX  (白)  ──────►  D3 (经分压)   SoftwareSerial TX

分压电路 (5V → 3.3V):
  D3 ──[1KΩ]──┬──[2KΩ]──► GND
               │
               └──────────► TM1026M RX (白线)
```
> 分压计算: 5V × 2K/(1K+2K) = 3.3V ✓
> TX/RX 不要接反: Nano D2 接模块 TX (绿), Nano D3 接模块 RX (白)。

## 4. EEPROM 数据布局 (1024 B)

```
0x000  4B   Magic 0xA5C30001  (校验是否已初始化)
0x004  1B   unlockCount low
0x005  1B   unlockCount high
0x006  1B   themeId
0x007  1B   silentMode
0x020  128B Users[8] 每个 16B
              [+0]  active=0xAA
              [+1..+9]  name (8 char + null)
              [+10..+12]  R/G/B
              [+13]  melodyId (0-3)
0x0A0  576B Messages[6] 每个 96B
              [+0]  active=0xAA
              [+1]  targetId (0=群发)
              [+2]  readFlag
              [+3]  flags (bit0=置顶)
              [+4..+5]  createdHour
              [+6..+7]  ttlHours (0xFFFF=永久)
              [+8..+87]  text (80B null-terminated)
0x2E0  4B   petXP / petLastInteractHour
0x2F8  60B  Log[10] 每个 6B (id + uptimeHour)
0x334  1B   logHead
0x352  1B   口令 active (0xAA=已录制)
0x353  100B 口令包络数据 (归一化 0-255, 每 30ms 一个点, 共 3 秒)
0x3B7-0x3FF reserved (~73B 余量)
```

总占用 ≈ 951B / 1024B (剩 73B 给未来扩展)。

## 4.5 口令波形匹配

### 原理

用 MAX9812 麦克风录制口令的**振幅包络** (amplitude envelope):
- 采集 100 个点, 每点 30ms, 覆盖约 3 秒
- 每个点取该 30ms 窗口内的最大 ADC 偏差
- 归一化到 0-255 (除以全局最大值 ×255), 消除音量差异
- 存入 EEPROM 作为模板

验证时再次采集, 与模板逐点比较:
```
score = Σ|captured[i] - template[i]| / 100
通过条件: score < 30 (阈值可改 VP_THRESH)
```

### 解锁流程变化

```
原: 声音唤醒 → 指纹验证
新: 声音唤醒 → 说口令 → 口令通过 → 指纹验证
              (无口令模板时跳过此步)
```

### 使用方法

```
串口 9600:
> vc          ← 录制口令 (倒计时 3 秒后对着麦克风说口令)
> vs          ← 查看口令状态 (ON/OFF)
> vd          ← 删除口令模板
```

### 注意事项

- 口令建议 2-3 个音节, 节奏清晰 (如"开门"、"芝麻开门")
- 每次录制会覆盖之前的模板
- 不同音量说同样口令能通过 (归一化)
- 换人说同样口令可能通过 (这不是声纹识别, 是口令节奏匹配)
- 用 `test_passphrase.ino` 可以独立测试匹配分数

## 5. 串口命令 (波特率 9600)

| 命令 | 说明 |
|:--|:--|

## 6. 桌面宠物 🐱

待机屏右下角有一只小猫:
- **happy** 默认状态, 睁着圆眼, 摇尾巴, 在屏幕右侧来回走
- **excited** 解锁后短暂触发, 星眼 + 心心泡泡
- **sad** 解锁失败 / 8 小时无人理触发, 耳朵下垂 + 流泪
- **sleepy** 24 小时没人理触发, 灰色 + Z 字, 静止不动

每次成功解锁 +10 经验, 经验存 EEPROM 重启不丢。

## 9. 快速上手

### Step 1 — 接线
按 §3 接线表组装。**最容易踩坑**: TFT 的 CS/DC/RST 是 A1/A2/A3, 不是 D 引脚 (Nano 的 A 脚也能当数字 IO)。

### Step 2 — 录 ISD1820 语音
单独给 ISD1820 接 5V/GND/喇叭, 按住 REC 录"欢迎回家"(≤10s), 松开 REC, 按 PLAYE 验证。

### Step 3 — 上传代码
1. Arduino IDE → 开发板: **Arduino Nano**, 处理器: **ATmega328P** (老板子选 Old Bootloader)。
2. 安装 §8 三个库。
3. 打开 `fingerprint_door_lock.ino`, 上传。
4. 期望开机看到: 扫描线 → 锁图标 → "Smart Door Lock" + 五声上行音, 然后进入 LOCKED 待机屏。

### Step 4 — 录入指纹
串口监视器, 9600:
```
> e
Enroll ID=1
> [放手指 → 移开 → 再放同一手指]
Enroll OK ID=1
```

### Step 5 — 配置个性化 + 留言
```
> u 1 Yang 0 200 80 1
> u 2 Mom  255 100 100 0
> m Welcome home everyone!
> m@1 Yang, your package arrived
```
重启后所有配置都还在 (EEPROM)。

### Step 6 — 录制口令 (可选)
```
> vc
Say in 3s...
[3 秒后对着 MAX9812 麦克风说口令, 如"开门"]
Saved!
> vs
Voice: ON
```
录制后, 解锁流程变为: 声音唤醒 → 说口令 → 口令通过 → 指纹验证。
不录制则跳过口令步骤, 与原来一样。

### Step 7 — 联调测试
1. 拍手 → 进入 WAKE 动画。
2. (如已录口令) 说口令 → 口令通过。
3. 已录入手指 → 个性化欢迎 + 留言弹出 + 5s 后自动锁。
4. 错指纹 3 次 → 暴力锁定 30s。
5. 拍手后 15s 不动 → 门铃模式。
6. `silent` 命令切静音。

## 11. 常见问题

| 现象 | 原因 / 排查 |
|:--|:--|
| TFT 全白 / 没图像 | 检查 CS/DC/RST 是否接到 A1/A2/A3; 电源 5V 是否稳定 |
| TFT 颜色反了 | 调用 `tft.invertDisplay(true)` (代码暂未启用) |
| 指纹模块"未找到" | TX/RX 是否交叉; D3 → RX 必须分压; 测量 3.3V 供电 |
| 不会被声音唤醒 | KY-038 板载电位器调灵敏度; 阈值改 60-150 |
| 编译报错 "out of memory" | 库没装全; 或 Nano flash 不够, 删掉部分动画 |
| 上传 Sketch too big | 同上, 优先删 `drawSparkle` 烟花和 sin/cos 动画 |
| 蓝灯有抖动 (老问题) | v2 已修复 (RGB_B 移到 D5) |
| RGB 颜色不对 | 共阳极板把 `RGB_COMMON_ANODE` 设 `true` |
| 留言不显示 | 检查 `ml` 是否有数据; 留言文本只支持 ASCII |
| 桌面宠物总是 sad | 没人解锁超过 8 小时; 解锁一次就会变回 happy |

## 13. 模块逐项测试 (独立烧录验证)

在组装完整系统之前, 建议逐个模块独立测试, 确认接线和功能正确后再合入主程序。

每个测试程序都是独立 sketch, 单独编译上传, 串口 9600 观察输出。

### 测试程序清单

| 目录 | 测试内容 | 需要的引脚 |
|:--|:--|:--|
| `test_tft/` | TFT 屏幕全部 UI 画面 (9 个界面循环) | A1, A2, A3, D11, D13 |
| `test_buzzer_led/` | 蜂鸣器全部旋律 + RGB LED 逐色 + 呼吸灯 | D5, D8, D9, D10 |
| `test_mic/` | KY-038 数字触发 + MAX9812 模拟波形柱状图 | D7, A0 |
| `test_isd1820/` | ISD1820 自动/手动播放 (每 5 秒一次) | D4 |
| `test_relay/` | 继电器自动通断 (每 2 秒切换, 听咔嗒声) | D6 |
| `test_fingerprint/` | TM1026M 检测/录入/匹配/清空/计数 | D2, D3 (分压) |
| `test_passphrase/` | 口令波形录制 + 匹配分数测试 | A0, D7 |

### 2. test_buzzer_led — 蜂鸣器 + RGB LED 测试

### 3. test_mic — 麦克风声音检测测试

```
KY-038 麦克风           Nano

DO               ──────► D7        数字输出, 有声音时 LOW
```

### 4. test_isd1820 — 语音录放模块测试

```

GND              ──────► GND
PLAYE            ──────► D4        边沿触发播放
```

> **先录音再测试**: 单独给 ISD1820 接 5V/GND/喇叭, 按住 REC 按键 → 对着模块麦克风说"欢迎回家"(≤10s) → 松开 → 按 PLAYE 验证有声 → 然后接 D4 上传测试程序。
> 测试内容: 每 5 秒自动播放一次, 串口发 'p' 手动触发。

### 5. test_relay — 继电器测试

```
继电器模块             Nano          电磁锁回路 (可选)
─────────             ───────       ──────────
VCC              ──────► 5V
GND              ──────► GND
IN               ──────► D6        高电平触发

COM              ──────► 12V 电源正极
NO               ──────► 电磁锁正极
(电磁锁负极)     ──────► 12V 电源负极
```

> 没有电磁锁也能测: 听继电器"咔嗒"声, 或万用表蜂鸣档测 COM-NO 通断。
> 电磁锁两端并联 1N4007 续流二极管 (阴极接正极)。
> 测试内容: 每 2 秒自动通断, 串口发 'o' 手动开 / 'c' 手动关。

### 6. test_fingerprint — 指纹传感器测试

```
TM1026M 指纹模块       Nano
───────────────       ───────
VCC (红)         ──────► 3.3V          ⚠️ 必须 3.3V, 接 5V 会烧!
GND (黑)         ──────► GND
TX  (绿)         ──────► D2            SoftwareSerial RX
RX  (白)         ──────► D3 (经分压)   SoftwareSerial TX

分压电路 (5V → 3.3V):
  D3 ──[1KΩ]──┬──[2KΩ]──► GND
               │
               └──────────► TM1026M RX (白线)
```

> **最容易踩坑的模块**:
> 1. TX/RX 不要接反: Nano D2 接模块 TX (绿), Nano D3 接模块 RX (白)
> 2. D3 输出 5V, TM1026M RX 只能 3.3V, 必须分压: 5V × 2K/(1K+2K) = 3.3V
> 3. 供电必须 3.3V (Nano 板载 3V3 引脚)
>
> 测试内容: 自动检测模块 → 显示已存指纹数 → 放手指自动匹配。
> 串口命令: 'e' 录入 / 'd' 清空 / 'c' 计数 / 'm' 手动匹配。

### 7. test_passphrase — 口令波形匹配测试

```
MAX9812 声波模块        Nano
───────────────        ───────
VCC              ──────► 3.3V      ⚠️ 不是 5V!
GND              ──────► GND
OUT              ──────► A0        模拟输入

KY-038 (可选, 用于快速触发)
──────────────────
VCC              ──────► 5V
GND              ──────► GND
DO               ──────► D7
```

> 测试内容:
> - 串口 'r' → 录制口令 (倒计时 3 秒后对着 MAX9812 麦克风说口令)
> - 串口 't' → 测试口令 (说口令, 显示匹配分数)
> - 串口 'd' → 删除模板
> - 串口 's' → 查看状态
>
> 口令建议 2-3 个音节, 节奏清晰 (如"开门"、"芝麻开门")。
> 匹配分数 < 30 为通过, 数字越小越严格。
> 不同音量说同样口令能通过 (归一化), 换人说可能偏高。

### 推荐测试顺序

```
1. test_tft         ← 先确认屏幕能亮
2. test_buzzer_led  ← 确认声音和灯光
3. test_mic         ← 确认麦克风能检测声音
4. test_isd1820     ← 确认语音播放 (先录音再测)
5. test_relay       ← 确认继电器吸合
6. test_fingerprint ← 测指纹 (需要 3.3V 分压, 最复杂)
7. test_passphrase  ← 测口令匹配 (需要麦克风)
8. 烧主程序 fingerprint_door_lock.ino ← 全部通过后合入
```
